# Provenance Guard — planning.md

## Architecture

```
POST /submit
  → validate input (text, creator_id)
  → Signal 1: Groq LLM score (0–1, likelihood of AI)
  → Signal 2: Stylometric score (0–1, sentence variance + punctuation density)
  → combined = (0.70 × llm_score) + (0.30 × stylo_score)
  → map score to label
  → write audit log entry
  → return { content_id, attribution, confidence, label }

POST /appeal
  → look up content_id in log
  → update status to "under_review"
  → append appeal_reasoning + timestamp to log entry
  → return confirmation

GET /log
  → return recent audit log entries as JSON
```

## Detection Signals

**Signal 1 — LLM classification (Groq)**
Asks the model whether the text reads as human or AI-generated. Returns a score 0–1. Captures semantic patterns: generic phrasing, smooth transitions, consistent register. Blind spot: formal human writing (academic, legal) scores high because it shares surface features with AI output.

**Signal 2 — Stylometric heuristics (pure Python)**
Measures two properties: sentence length variance and punctuation density. AI text tends to be structurally uniform; human writing is more erratic. Returns a score 0–1. Blind spot: poetry and non-native English speakers writing carefully can appear AI-like to this signal.

## Uncertainty Representation

```
combined = (0.70 × llm_score) + (0.30 × stylo_score)

< 0.45  → Likely Human-Written
0.45–0.75 → Uncertain
> 0.75  → Likely AI-Generated
```

Thresholds are asymmetric on purpose: the bar to label something AI (>0.75) is higher than the bar to label it human (<0.45). A false positive — wrongly flagging a human writer — is worse than missing an AI piece, so the system requires stronger evidence before making that call.

A score of 0.6 means the system genuinely can't tell. The label reflects that honestly rather than forcing a binary.

## Transparency Label Variants

**High-confidence AI (> 0.75)**
> "Attribution unclear — Our automated analysis found patterns consistent with AI-generated text. This is a signal, not a certainty. Use the appeal button below to contest this."

**Uncertain (0.45–0.75)**
> "Attribution uncertain — Our analysis could not determine whether this was written by a person or generated by AI. This flag is for context, not judgment."

**Likely human (< 0.45)**
> "Appears human-written — Our analysis found no strong indicators of AI generation. This reflects our best assessment, not a guarantee."

## Appeals Workflow

Any creator can appeal using the `content_id` from their submission. They provide a `creator_reasoning` field explaining why they think the classification is wrong. The system updates their content's status to `under_review`, appends the reasoning and timestamp to the audit log, and returns a confirmation. No automated re-classification — a human reviewer handles it from there.

## Anticipated Edge Cases

1. **Formal poetry** — repetitive structure and simple vocabulary make stylometrics score it as AI-like, even when it's genuinely human. The LLM signal may correct this, but it's a known weak spot.
2. **Non-native English speakers** — careful, uniform writing style can resemble AI output structurally. The asymmetric threshold and appeals workflow exist specifically for this case.
